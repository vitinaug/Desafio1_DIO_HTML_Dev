O que são Clients?

Voltar

O Client sempre inicia a comunicação com o servidor. O mais conhecido dele é o navegador.

Quando é preciso acessar um site, uma requisição é disparada pela internet para o computador no qual o site está hospedado ( servidor ).

Após enviar uma requisição, o cliente aguarda até que haja uma resposta do servidor e então a comunicação é encerrada.

Ao trabalhar com o client-side, temos que ter algumas preocupações extras, como responsividade e compatibilidade com navegadores.

Navegadores seguem o padrão W3C, mas sempre há diferenças entre eles.

  • Clique no link para entrar no site que nos mostra a compatibilidade entre tags HTML e os navegadores.

O JavaScript pode ser utilizado no client-side, pois os navegadores possuem um módulo ( V8 ) capaz de interpretar a linguagem

O que são Servers?

Voltar

O Server recebe uma requisição e devolve uma resposta para o cliente.

O servidor é um ou mais computadores com um único endereço. Neste computadores podem ser hospedados documentos, imagens, áudios, etc.

Linguagens usadas no server-side:

  • JavaScript
  • PHP
  • C#
  • Ruby
  • Entre outras!

Como é uma estrutura básica de um HTML?

Voltar

Um documento HTML é constituido principalmente por duas partes:

Head: É o cérebro da página web. Não é visível para o usuário, mas contém informações cruciais sobre a página, como metadados, links e configurações de estilo. É carregado antes do conteúdo da página em si.

Body: Engloba todo o conteúdo visível da página. É aqui que inserimos textos, imagens, links, formulários que os visitantes verão e irão interagir.

Listas ordenadas vs. Listas não ordenadas

Voltar

Listas Ordenadas:

Lista numerada em ordem crescente. Tem várias finalidaes, uma receita por exemplo, em que a ordem importa para que a refeição seja feita da maneira correta.

No documento HTML, a tag utilizada para está lista é ol.

Exemplo:

  1. Quebrar 3 ovos.
  2. Pega uma xícara de óleo
  3. Bater no liquidificador.

Listas Não Ordenadas:

Lista não numerada. Tem várias finalidades também. Construir um menu é uma delas.

No documento HTML, a tag utilizada para está lista é ul.

Exemplo:

  • Jogar
  • Configurações
  • Sair

Tags e Atríbutos

Voltar

Tags: Funcionam como instruções que ditam a estrutura e comportamento de uma página web.

Algumas Tags:

  • title: Define o título da página que é exibida na barra de título do navegador. Parte vital para otimização de mecanismos de buscas.
  • i: Deixa o texto em itálico.
  • strong: Deixa o texto em negrito.
  • u: Define o texto sublinhado.
  • h#: # = número. Títulos e Subtítulos. Quando maior o número, menor sua hierarquia.
  • p: Define um paragrafo.
  • blockquote: Define uma citação.
  • mark: Marca o texto.
  • sup: Define o texto como sobrescrito.
  • a: Cria um link para uma outra página.
  • hr: Insere uma linha horizontal na página.

Atríbutos: São propriedades que uma tag pode ter. Existem os atríbutos mais gerais e as específicas de tags.

Alguns Atríbutos:

  • id: Identificador de elemento. Útil ao usar JavaScript.
  • style: Serve para colocar comandos CSS diretamente na tag.
  • class: Serve para padronizar as tags de acordo com uma classe. Utilizado no CSS.
  • type: Altera o tipo de entrada. Tag específica: input
  • src: Caminho até a imagem que queremos inserir na página. Tag específica: img
  • width: Altera a largura da imagem. Tag específica: img
  • href: Referência o destino do link. Tag específica: a
  • target: Define se o link será aberto na mesma aba ou em uma outra. Tag específica: a
